## Summary

Release v1.7.1 focuses on reliability and release hygiene rather than new
end-user features. Across the `v1.7.0...v1.7.1` changelog, the team extracted
and consolidated AppHost database-clear behavior, fixed CI lint breakages, and
aligned conflict-prone areas so ongoing delivery can stay fast and predictable.

Release v1.7.1 is live at
[github.com/mpaulosky/MyBlog/releases/tag/v1.7.1](https://github.com/mpaulosky/MyBlog/releases/tag/v1.7.1).

## What's New

### AppHost command extraction and cleanup

Multiple PRs in this release range reinforce the same direction: centralizing
MongoDB AppHost command wiring in extension methods to reduce duplication and
lower null-reference risk in startup orchestration.

```csharp
// AppHost composition now favors extension-based wiring
builder.AddMongoDB("mongodb")
    .WithClearDatabaseCommand();
```

This keeps development utilities explicit and easier to reason about when
running local workflows.

### CI YAML lint hardening

The release includes a dedicated YAML lint configuration to unblock and
stabilize the `lint-yaml` pipeline step.

```yaml
# .yamllint.yml (added in v1.7.1 range)
extends: default

rules:
  line-length:
    max: 160
```

The practical outcome is fewer false-positive lint failures and more consistent
CI feedback for workflow edits.

## How to Upgrade

No database migrations or breaking configuration changes are required for this
point release.

1. Pull `main` and update to the `v1.7.1` tag.
2. Restore dependencies and rebuild.
3. Run the solution test matrix used by this repo (unit, architecture, bUnit,
   integration, AppHost).
4. If you customize workflow lint rules, keep local YAML rules aligned with
   `.yamllint.yml`.
